## 二、9.2 目标分项达标路径

### 1. 代码质量与架构 (72 → 92)

**当前问题：** 无 TypeScript、无 ESLint、无测试、connectionStore 过大
**达标方案：**
- 迁移至 TypeScript（渐进式，3 周）：先为核心 store/model 添加 `.ts`，再扩展至组件
- 配置 ESLint + Prettier（1 天）：airbnb + vue3 规则集，统一代码风格
- 引入 vitest + @vue/test-utils（2 周）：store 单元测试覆盖 > 80%、组件交互测试 > 60%
- 拆分 connectionStore（2 天）：拆为 `authStore` / `sessionStore` / `connectionStore`，单文件 < 150 行
- Git Hooks（半天）：husky + lint-staged，提交前自动 lint + test

### 2. UI/UX 设计 (78 → 94)

**当前问题：** 无 SFTP UI、无拖拽标签、无分屏、无动画主题切换
**达标方案：**
- **SFTP 浏览器增强**（2 周）：树形 + 列表双视图，拖拽上传/下载，右键菜单，进度条，sudo 支持
- **终端分屏**（1 周）：水平/垂直分屏（split-pane），最多 4 面板，拖拽调整大小
- **标签拖拽排序**（3 天）：HTML5 drag & drop API，标签右键菜单（关闭其他、重命名、复制会话）
- **动画系统**（2 天）：页面过渡 + 列表入场 + 通知弹出 + 主题切换过渡动画，prefers-reduced-motion 支持
- **主题系统增强**（3 天）：自定义颜色选择器，导入/导出主题，社区主题市场
- **字体连字支持**（1 天）：Fira Code / JetBrains Mono 连字开关
- **图标系统统一**（1 天）：所有操作均带图标，统一的 16/20/24 三级尺寸体系

### 3. 功能完整度 (60 → 92)

**当前问题：** 无法 SFTP、无端口转发、无会话持久化、无搜索
**达标方案：**
- **SFTP 完整实现**（3 周）：
  - Node.js 后端：ssh2 SFTP 模块 + 文件流式传输
  - 前端：文件树 + 编辑器 (Monaco) + 图片预览 + 视频/音频播放
  - 服务端对服务端文件传输
- **端口转发**（2 周）：本地/远程/动态 SOCKS5 转发，健康监控，自动重连
- **会话持久化**（1 周）：IndexedDB 存储会话历史 + 浏览器关闭后恢复
- **终端搜索**（3 天）：Ctrl+F 搜索终端输出，高亮匹配
- **命令自动补全**（3 天）：基于历史命令的模糊补全
- **Session 录制与回放**（2 周）：基于 xterm addon 的录制，可回放查看
- **多主机命令执行**（1 周）：选中多台服务器，一键广播命令
- **SSH 配置导入/导出**（2 天）：OpenSSH config 格式兼容，加密导出

### 4. 性能与构建 (71 → 90)

**当前问题：** 无代码分割、无 PWA、完整 Bulma CSS
**达标方案：**
- **选择性 Bulma 导入**（2 天）：仅 import 使用的组件，减少 70% CSS 体积
- **PWA 支持**（3 天）：Service Worker + manifest.json + 离线回退页面
- **路由级 + 组件级代码分割**（2 天）：`defineAsyncComponent` 拆分大组件
- **Bundle 分析**（半天）：配置 `rollup-plugin-visualizer`，设定性能预算（初始 JS < 200KB）
- **图片优化**（1 天）：SVG 精灵图 + 懒加载
- **字体优化**（1 天）：Inter + Fira Code 子集化，`font-display: swap`

### 5. 安全性 (75 → 95)

**当前问题：** 无主密码、无 HTTPS 强制、无安全审计
**达标方案：**
- **主密码锁**（1 周）：首次使用设置主密码，加密所有凭证存储，闲置自动锁定
- **HTTPS 强制**（1 天）：配置指南 + 自动 Let's Encrypt 脚本
- **安全审计日志**（1 周）：所有认证/连接/文件操作记入审计日志，可导出
- **XSS 防护加固**（2 天）：DOMPurify 处理所有用户输入显示，加强 CSP
- **速率限制**（1 天）：登录失败 + 连接失败速率限制，防止暴力破解

### 6. 可维护性 (50 → 90)

**当前问题：** README 一句话、无测试、无 CI/CD
**达标方案：**
- **完善文档体系**（1 周）：
  - `ARCHITECTURE.md`：架构图 + 数据流 + 模块职责
  - `DEVELOPMENT.md`：环境搭建 + 编码规范 + 测试指南
  - `DEPLOYMENT.md`：Docker/Nginx/PM2 部署方案
  - `CONTRIBUTING.md`：PR 流程 + commit 规范
- **CI/CD 流水线**（2 天）：GitHub Actions → lint → test → build → deploy
- **API 文档**（2 天）：JSDoc → TypeDoc 自动生成
- **错误监控**（2 天）：全局错误边界 + Sentry 集成
- **Storybook 组件目录**（1 周）：所有组件独立预览 + 交互测试

### 7. 响应式/无障碍 (50 → 85)

**当前问题：** 无 focus-trap、无 aria-live、无 skip-to-content
**达标方案：**
- **焦点管理**（2 天）：所有弹窗/侧栏 focus-trap，Tab 键顺序合理
- **ARIA 增强**（3 天）：`aria-live="polite"` 通知区域、`aria-expanded` 侧栏、`role="tablist"` 标签栏
- **Skip to content**（半天）：页面顶部跳过导航链接
- **高对比度模式**（2 天）：`prefers-contrast: more` 媒体查询 + 单独高对比主题
- **WCAG 2.1 AA 验证**（2 天）：axe-core 自动化检测 + 手动验证
- **多语言**（2 周）：vue-i18n + 中/英/日/韩语言包

## 五、风险与缓解

| 风险 | 概率 | 影响 | 缓解方案 |
|---|---|---|---|
| ssh2 在 Node.js WebSocket 场景的性能瓶颈 | 中 | 高 | 采用 worker_threads 隔离 SSH 连接，每连接独立线程 |
| SFTP 大文件传输的 WebSocket 内存压力 | 中 | 高 | 流式传输分块（chunked），每块 < 64KB，内存池复用 |
| CSP 兼容性导致第三方服务异常 | 低 | 中 | 严格测试，提供 CSP 宽松模式开关（警告提示） |
| 主密码遗忘导致数据不可恢复 | 低 | 高 | 加密导出 + 密码提示 + 生物识别（WebAuthn）备用 |
| TypeScript 迁移期间编译错误 | 中 | 中 | 渐进式迁移，`any` 白名单过渡，`strict: false` 启动 |

## 六、结论

**Haossh 目标 9.2/10 完全可行。** 主要竞争差异点：

1. **Web 原生 + 全平台**（vs WinSSH 仅桌面）
2. **极致 UI/UX**（vs Teleport 功能化、Guacamole 过时）
3. **企业级安全**（AES-256-GCM + CSP + 主密码锁 + 审计日志）
4. **轻量一键部署**（vs Teleport 重量级、Guacamole Java 栈）
5. **无障碍国际标准**（WCAG AA + i18n，vs 所有竞品）

总计约 **14 周**可实现目标。请确认后开始 Phase 1 代码编写。
