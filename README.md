## 🚀 一键部署

### 方法一：一键安装脚本（推荐）

```bash
curl -fsSL https://raw.githubusercontent.com/guoxpeng/webssh/main/deploy.sh | bash
```

一键完成：下载代码 → 安装依赖 → 构建前端 → 启动服务

然后在浏览器打开 `http://你的服务器IP:9627`

### 方法二：Docker 部署

```bash
docker run -d --restart=unless-stopped --name webssh -p 9627:9627 \
  -e PORT=9627 \
  node:20-alpine sh -c "apk add git openssh && \
  git clone --depth=1 https://github.com/guoxpeng/webssh.git /app && \
  cd /app && npm install && npm run build && node server/index.mjs"
```

### 方法三：手动部署

```bash
# 1. 下载代码
git clone https://github.com/guoxpeng/webssh.git
cd webssh

# 2. 安装依赖
npm install

# 3. 构建前端
npm run build

# 4. 启动服务（单端口运行，前端后端合一）
PORT=9627 node server/index.mjs
```

然后在浏览器打开 `http://localhost:9627`

### 开发模式

```bash
# 终端1：启动后端
node server/index.mjs

# 终端2：启动前端开发服务器（热更新）
npm run dev
```

开发模式前端地址 `http://localhost:5173`，后端 `http://localhost:9627`。

## 📖 怎么使用？

### 第一步：设置主密码 🔐

第一次打开网页，会让你设置一个**主密码**。这个密码用来加密你保存的所有服务器密码。

> 就像手机上的锁屏密码，别人拿到你的电脑也看不到你的密码！

### 第二步：添加服务器 🔗

1. 点左边菜单的"服务器"
2. 填上：
   - **名称** — 给服务器起个名字（比如"我的博客服务器"）
   - **主机/IP** — 服务器地址（比如 192.168.1.100）
   - **端口** — 默认 22（不用改）
   - **用户名** — 登录用的账号（比如 root）
   - **密码或私钥** — 登录的钥匙
3. 点击"连接"就开始远程管理了！

### 第三步：多标签管理 🏷️

- 每台服务器独立标签，标签栏显示 `user@host`
- 点击标签切换不同终端
- 关闭标签不影响其他连接
- 从服务器列表再次连接服务器，会在旁边新增标签

### 第四步：开始用吧！🎉

连上之后你会看到一个**黑底白字的窗口**，这就是服务器的命令行。

## 功能速览 📋

### 🖥️ 终端操作
连上服务器后看到的命令行窗口，可以输入各种命令。

### 📁 文件管理
在服务器上浏览、上传、下载文件，像用电脑文件管理器一样。

### 🚇 SSH 隧道
给数据挖一条"秘密通道"，安全访问远程网络。

### 🐳 Docker 管理
如果你的服务器装了 Docker，可以在网页上查看容器状态、启动、停止、看日志。

### 📝 命令片段
把常用的命令存起来，**一键发送**到终端，不用反复输入。

### 💾 备份管理
备份所有服务器的配置，换电脑了**一键恢复**，什么都不丢。

### 🌏 中英文切换
设置面板里点一下，界面就在中文和英文之间切换。

### 🎨 主题换肤
内置 4 种颜色主题：浅色、深色、紫色（Dracula）、北欧风。
